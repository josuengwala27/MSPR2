# Gestion de Projet et Organisation Agile

## 1. Contexte et Objectifs
- Piloter un projet complexe (data, IA, frontend, API) pour l’OMS, avec des exigences de qualité, d’accessibilité et de transparence.
- Impliquer toutes les parties prenantes (techniques, métiers, utilisateurs finaux).

## 2. Méthode de Gestion
- **Méthode agile (Scrum/Kanban)** : itérations courtes, feedback continu, adaptation rapide.
- **Roadmap évolutive** : vision à moyen terme, ajustée selon les retours et priorités OMS.
- **Jalons clairs** : validation à chaque étape clé (modélisation, API, frontend, tests, documentation).

## 3. Roadmap et Jalons

| Phase                | Objectifs principaux                        | Jalons/Deliverables                |
|----------------------|---------------------------------------------|------------------------------------|
| 1. Initialisation    | Analyse besoins, choix techno, maquettes    | Docs conception, choix, ERD        |
| 2. Data/ETL/BDD      | Extraction, nettoyage, modélisation         | Scripts ETL, schéma BDD, tests     |
| 3. API & IA          | API Express, API IA, modèles ML             | Endpoints REST, modèles, tests ML  |
| 4. Frontend          | UI accessible, visualisation, intégration   | Dashboard, pages prédiction, tests |
| 5. Documentation     | Livrables, guides, benchmarks, accessibilité| README, docs, OpenAPI, guides      |
| 6. Déploiement       | Mise en production, support, formation      | Déploiement, support, formation    |

## 4. Backlog et Suivi
- **Backlog** : liste priorisée des tâches (features, bugs, docs, tests), évolutive selon les retours.
- **Outils** : GitHub Issues, Kanban board, todo list collaborative.
- **Suivi** : points d’avancement hebdomadaires, reporting synthétique (tableaux, graphiques, burndown).

## 5. Reporting et Communication
- **Rapports réguliers** : avancement, risques, décisions, points d’attention.
- **Démos** : présentation des fonctionnalités à chaque jalon.
- **Documentation vivante** : mise à jour continue des docs et guides.

## 6. Adaptation Continue
- **Rétrospectives** : analyse des succès/difficultés à chaque fin de phase.
- **Adaptation** : ajustement de la roadmap, du backlog, des priorités selon les retours OMS et utilisateurs.
- **Veille** : intégration des évolutions technologiques et réglementaires.
